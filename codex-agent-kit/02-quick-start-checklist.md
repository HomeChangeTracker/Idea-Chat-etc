# 빠른 시작 체크리스트 (프로젝트 시작 전 / AGENTS.md 생성 전)

아래는 설문지를 다 채우기 전에 최소한으로 결정해야 할 항목입니다.

## A. 프로젝트 방향

- [ ] 이 프로젝트의 핵심 목표를 1~3개로 적었다
- [ ] 이번에 하지 않을 것(Non-goals)을 적었다
- [ ] MVP 기준(언제 "일단 출시/완료"인지)을 적었다

## B. 기술 기본값

- [ ] 기본 스택(프론트/백엔드/DB)을 정했다
- [ ] 패키지 매니저를 정했다
- [ ] 린트/테스트/빌드 명령을 정했다
- [ ] 피하고 싶은 기술/도구를 적었다

## C. 작업 규칙

- [ ] 브랜치 전략을 정했다
- [ ] 커밋 메시지 규칙을 정했다
- [ ] 의존성 추가 시 확인 규칙을 정했다
- [ ] 문서 업데이트 규칙을 정했다

## D. Codex 협업 방식

- [ ] Codex가 질문해야 하는 상황을 적었다
- [ ] Codex가 자율로 진행 가능한 범위를 적었다
- [ ] Codex 금지사항을 적었다
- [ ] 내가 자주 요청할 작업 유형을 적었다

## E. 예외 규칙 분리 여부

- [ ] `frontend/`, `backend/`, `infra/` 등 하위 디렉터리별 규칙 분리가 필요한지 판단했다
- [ ] 프로젝트 루트 `AGENTS.md`만으로 충분한지 판단했다

---

## 최소 입력만으로 AGENTS.md 만들기 (초간단 버전)

아래 8개만 적어도 Codex에게 초안을 만들게 할 수 있습니다.

- 프로젝트 종류
- 핵심 목표
- Non-goals
- 기본 스택
- 검증 명령(lint/test/build)
- 브랜치/커밋 규칙
- Codex 자율 범위
- Codex 금지사항
