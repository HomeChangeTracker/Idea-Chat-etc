# Agent Skill 후보 정리 템플릿 (반복 작업 분리용)

이 문서는 "어떤 작업을 Skill로 뺄지" 정리하는 용도입니다.
처음에는 지침-only 스킬로 시작하고, 정말 반복되는 경우에만 `scripts/`를 붙이는 것을 권장합니다.

## 스킬 선정 기준

- 작업이 반복되는가? (월 2회 이상)
- 단계가 비교적 고정적인가?
- 입력/출력이 명확한가?
- 실수 비용이 높은가? (예: 배포 전 점검, 보안 체크)

## 추천 시작 스킬 (개인 프로젝트 기준 예시)

- `feature-planning`: 기능 요청을 구현 가능한 TODO/수용 기준으로 정리
- `code-review-rules`: 프로젝트 규칙 기반 리뷰 체크리스트 적용
- `release-check`: 배포 전 검증 순서 점검
- `bug-triage`: 버그 리포트 정리 및 재현 조건 명확화

## Codex에 스킬 설계 요청할 때 쓰는 문장

```md
아래 반복 작업을 Agent Skill로 만들고 싶어.

작업:
- [작업명]

요구사항:
- trigger 조건과 비-trigger 조건을 분명히 작성
- 지침-only 스킬로 먼저 설계
- 필요하면 나중에 scripts 추가 포인트도 적기
- `SKILL.md` 초안까지 작성

[내 프로젝트 규칙 / AGENTS.md / 예시 입력]
```
