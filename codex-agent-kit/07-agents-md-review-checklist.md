# AGENTS.md 품질 점검 체크리스트 (생성 후 검토용)

`AGENTS.md`를 만든 뒤 아래 항목으로 점검하면, Codex가 실제 작업에서 덜 헷갈립니다.

## 1) 실행 가능성

- [ ] "잘 해라" 같은 추상 문장이 아니라 행동/명령 기준으로 적혀 있다
- [ ] `lint`, `test`, `build` 명령이 실제 프로젝트 명령으로 적혀 있다
- [ ] 의존성 추가 / 스키마 변경 / 운영 변경에 대한 승인 규칙이 있다

## 2) 충돌/중복

- [ ] 같은 주제의 규칙이 중복되지 않는다
- [ ] 서로 충돌하는 규칙(예: 자율 진행 vs 반드시 질문)이 없다
- [ ] 루트 규칙과 하위 override 규칙의 역할이 분리되어 있다

## 3) 컨텍스트 적합성

- [ ] 이 프로젝트에 필요 없는 일반론이 과하게 들어가 있지 않다
- [ ] 현재 스택/협업 툴/검증 방식에 맞게 적혀 있다
- [ ] 향후 바뀔 가능성이 큰 값은 예외 파일 또는 문서로 분리했다

## 4) Codex 협업 품질

- [ ] Codex 자율 범위가 명확하다
- [ ] Codex가 먼저 질문해야 하는 조건이 명확하다
- [ ] Codex 금지사항이 구체적이다
- [ ] 리뷰/구현/리팩터링 요청 시 기대 응답 스타일이 드러난다

## 5) 유지보수성

- [ ] 파일 길이가 너무 길지 않다 (핵심 규칙 중심)
- [ ] 프로젝트가 커지면 어느 디렉터리에 `AGENTS.override.md`를 둘지 기준이 있다
- [ ] 변경 시 누가/언제 업데이트할지 정해져 있다 (본인 프로젝트면 본인 루틴)
