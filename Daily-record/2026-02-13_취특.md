## 14:00 — 멘토링 진행

### 1) MVP 구현 가능성
- **멘토 문의:** 현재 설계한 MVP 구현 가능하다고 생각하는지?
    - _if possible_ → **MVP 기능은 모두 구현해야 함**

### 2) DB 방향
- **추천:** DB는 **PostgreSQL** 권장
    - **NoSQL:** 동적 정의 → 데이터 구조 유연하게 변경 가능
    - **PostgreSQL:** 정적 기반 + 동적 작업 용이, 자체 기능 지원 다양  
        → **장기 개발 관점에서 PostgreSQL 추천**

### 3) 개발 접근 방식 (2가지 관점 병행)
- **전체 구조 우선**
    - 전체 파이프라인 구성 후 테스트 진행 _(결과가 나오는지 확인)_
- **기능별 특화**
    - 각 기능을 별도로 고도화 _(피처 엔지니어링)_
- **이유**
    - 순차 완성 시 연결 구간에서 특이사항 발생 가능
- **우려사항**
    - 단계적 진행으로 문제 지점이 불명확할 수 있음  
        → 디버깅 용이하도록 설계/로그/검증 고려 필요

### 4) 기획서 관련
- 기능 구현이 가능하다면 **기획서 방향도 괜찮음**
- 현재 기획서는 **개발자 기준 경향**이 강함  
    → 실제 사용자(도구 사용자의 편의성) 관점도 반영 필요

### 5) 기술 스택/알고리즘 분해
- “모델이 알아서 다 한다” 방식보다
    - **정합, 세그멘테이션(seg) 등 기술 요소 분할** 필요
- **결정 필요**
    - 딥러닝 모델 사용 vs 전통적 알고리즘 사용
- 여러 알고리즘을 쓰는 만큼 **원리 이해 + 정확한 사용** 중요

### 6) 품질 체크
- 도출 결과가 **원하는 방향으로 나오는지 수시 점검** 후 다음 단계 진행

### 7) 프론트 진행 제안
- **기술 완성 → 사용자 편의성 고려** 순서로 프론트 진행 권장

### 8) 팀프로젝트 운영 팁
- 분산된 인원이 작업하므로 **코드 컨벤션 먼저 합의** 권장
- GitHub 머지/조정 편의 위해 **개발 리더를 정해 머지 담당** 고려
- 절대 규칙은 아니며, **개발 편의성 중심으로 운영** 권장
